1. Package overview

1.1 ParaDe packages

Location                               Description
org.makumba.parade.access              The classes that handle access to the ParaDe server
                                       (logging to the database, triggering of access control)
org.makumba.parade.auth                The classes for access control (HTTP Authorizer and
                                       LDAP implementation)
org.makumba.parade.aether              The implementation of Aether interfaces for ParaDe,
                                       and other classes needed for the Aether engine
                                       integration
org.makumba.parade.controller          The controller classes (Struts Actions and ParaDe
                                       controller classes)
org.makumba.parade.init                The classes needed for initialisation, amongst which
                                       the InitServlet (ran at start-up) and the classes that
                                       read configuration files
org.makumba.parade.listeners           ParaDe listeners, such as the CVS commit listener
org.makumba.parade.model               The ParaDe model classes and Hibernate mappings
org.makumba.parade.model.interfaces    The interfaces for model managers
org.makumba.parade.model.managers      The implementation of model managers
org.makumba.parade.tools               Various utility classes, amongst which the
                                       TriggerFilter that filters each access to ParaDe
org.makumba.parade.view                The servlets used to display the main views
org.makumba.parade.view.interfaces     The interfaces for view managers
org.makumba.parade.view.managers       The view managers, populating the templates
org.makumba.parade.view.templates      The Freemarker templates used to display the ParaDe
                                       view
org.makumba.parade.view.beans          A number of java beans used by the JSPs to render the view
webapp                                 Various JSPs for displaying pages
webapp/admin                           JSPs for ParaDe configuration
webapp/WEB-INF/classes/dataDefinitions The Makumba Data Definitions that make it possible
                                       to use Makumba with ParaDe

1.2 Aether packages

Location                               Description
org.makumba.aether                     The main classes of the Aether engine
org.makumba.aether.model               The Ather model classes and Hibernate mappings,
                                       including the classes of the percolator
org.makumba.aether.percolation         The classes that handle Aether event percolation
webapp/aether                          The JSPs used to configure the Aether engine at
                                       runtime
webapp/WEB-INF/classes/dataDefinitions The Makumba Data Definitions that make it possible
                                       to use Makumba with ParaDe

2. MVC components

ParaDe uses Hibernate in order to drive the model, and Struts as a controller. Aditionally it also uses Makumba in order to render the view, together with a number of Java beans.
